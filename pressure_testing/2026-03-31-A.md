# gzb one 框架性能压测报告 / Performance Benchmark Report

### 🎯 测试目标 / Test Objectives
寻找 **gzb one** 全栈框架调度开销与原生 Netty 直接响应的性能差值。
Measure the performance gap between the **gzb one** full-stack framework dispatching overhead and native Netty direct response.

> **核心逻辑 / Core Logic**: 由于本框架为全栈实现，Netty 处理器（Handler）之后的所有步骤均计为框架耗时。  
> As a full-stack implementation, all steps following the Netty Handler are calculated as framework overhead.

---

* **CPU**: AMD Ryzen 5 3600 (6 Core 12 Thread)
* **内存 (RAM)**: 32GB
* **操作系统 (OS)**: Windows 10

  Both Server and Benchmarking tool (wrk) run within the same VM. Single-thread and dual-thread server configurations are used to minimize resource contention.

### 📝 关键点说明 / Key Caveats

1. **环境波动 (Environment Fluctuations)**:  
   本次测试基于 VMware 虚拟化环境，存在固有波动。所有指标均为连续压测五次取峰值（非极端值，可复现）。  
   Tested on VMware with inherent jitter. Results represent the peak of 5 consecutive runs (reproducible, non-outlier).

2. **关于超时 (Regarding Timeouts)**:  
   在压测过程中，`wrk` 报告的超时数约等于并发连接数。通过自研工具 `gzb.frame.DDOSV2` 验证，未发现实际丢包，推测为 `wrk` 在高吞吐结束时的连接回收机制所致。  
   `wrk` reported timeouts roughly equal to the number of concurrent connections. Verification via `DDOSV2` confirmed no actual packet loss; this is likely due to `wrk`'s internal socket recycling mechanism under extreme throughput.

3. **调度损耗分析 (Dispatching Analysis)**:  
   `/test/users/2` 在逻辑复杂度高于 `/users/1` 的情况下 QPS 表现相近甚至略高，进一步证明当前瓶颈在于虚拟化环境的 I/O 调度与宿主机 CPU 争抢，而非框架内核逻辑。  
   The fact that `/test/users/2` matches or exceeds `/users/1` despite higher complexity proves the bottleneck lies in VM I/O scheduling and Host CPU contention, not the framework core.

public static FullHttpResponse response=null;
static{
    response = new DefaultFullHttpResponse(
            HttpVersion.HTTP_1_1,
            HttpResponseStatus.OK,
            Unpooled.wrappedBuffer(NettyTools.HELLO_WORD),
            false
    );
    HttpHeaders headers = response.headers();
    headers.set(HttpHeaderNames.CONTENT_TYPE, NettyTools.CONTENT_TYPE);
    headers.set(HttpHeaderNames.CONTENT_LENGTH, NettyTools.CONTENT_LENGTH);
    //headers.set(HttpHeaderNames.SERVER, NettyTools.SERVER_NAME);
    //headers.set(HttpHeaderNames.DATE, NettyTools.THIS_TIME);
}
protected void channelRead0(ChannelHandlerContext ctx, FullHttpRequest req) throws Exception {
    if (req.uri().equals("/text")) {
        ctx.write(response.retainedDuplicate());
    } else {
        Request request = new RequestDefaultImpl(ctx, req);
        Server.factory.start(request, request.getResponse());
    }
    ctx.flush();
}
